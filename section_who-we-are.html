UPcload / FitAnalytics

Developed for UPcload a parser that pre-processes the content of thousands of PDF and Excel files into a normalized database.

This parser saved hundreds of man hours that would have been spent manually collecting this data.

A Node.js backend with supporting Bash scripts was developed to perform this process. A Javascript/JQuery frontend was developed to access the data.

  • backend
  • infrastructure
  • supporting tools
  • pdf parser
  • webcam tool / flex frontend

Fraunhofer

I worked on these projects:

  • a semantic web server (written in Java, XML) modelling the underground traffic of Tokyo in real-time (each agent coresponds to a real passenger)
  • an ad-hoc hitchhiking platform (J2ME, C#, SQL Server, ASP.NET)
  • a mobile peer-to-peer media streaming app (J2ME)
  • a mobile plug-in oriented location based services composition app, with a map and POIs and Apps on that map (Java Beans and Dojo/JavaScript)

m-click

So at UPcload on average we have like 20k rpm (requests per minute) and on peak it goes up to 50rpm, average response time was an amazing 30-50ms after we did a lot of optimising. The garment data is quite complex so we used Redis to cache objects, one product had a garment and a mannequin so a request would query 10 tables. There's a MySQL cluster we set up, with mostly read-only slaves because the entire app was very read-heavy with few writes. The sever farm is a bunch of nodejs-clusters, performing very well (low CPU < 10% even on load) on Heroku and on DigitalOcean.

Here's UPcload in Action: At Zalando (click on 'Welche Größe passt mir?'): http://www.zalando.de/yumi-blazer-navy-yu121g000-k11.html

At The North Face (click on 'Size Advisor'): http://www.thenorthface.com/catalog/sc-gear/womens-shirts-tops-filter-category-tops/women-8217- s-short-sleeve-willow-green-shirt.html?variationId=H1H&variationName=FANFARE%20GREEN

It's also being tested in Russia (otto.ru) and in Brazil (http://www.dafiti.com.br/) and will be live there soon.

The webcam part of UPcload is used by e-tailors: http://webcam.upcload.com/upcload used all over the world (e.g. in South Africa and Japan). There's bunch of image processing servers deployed that act as a worker queue to do the number crunching: up to 100% cpu load each time body dimensions are calculated from webcam images.

Then there's m-click.aero, a set of high-reliability and bug-free management tools for flight controllers. It's about managing air spaces, booking missions and checking with business rules, e.g. whether certain flight routes are legal. The DB is Postgres, Back-end ist partly Java (for rules processing, XML) and PHP (for making DB requests) using Zend and Laravel. Front-end is JavaScript (ExtJS) with complex data-centric user interfaces. The software must not have any bugs once it's in production and the servers must never be down. Currently I'm working on a connection between Flight Control Stations and the air space management software. The part that speaks the proprietary Flight Control communication protocol is written in Java, the Rest is Postgres and Zend.
